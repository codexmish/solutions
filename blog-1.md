# TypeScript: any বনাম unknown 

## কেন any বিপজ্জনক?
any হলো টাইপ সেফটির একটি "গর্ত"। এটি ব্যবহার করলে `TypeScript` কোনো টাইপ চেক করে না ঠিক `JavaScript` এর মতো। ফলে কোডে ভুল থাকলেও এরর দেখায় না, যা পরে রানটাইমে অ্যাপ ক্র্যাশ করায়।



```
let value: any = 10;


value.toUpperCase(); // ভুল, কিন্তু TS কোনো এরর দিবে না। ফলে রানটাইমে ক্র্যাশ করবে।```


## কেন unknown নিরাপদ?
unknown টাইপ যেকোনো ভ্যালু নিতে পারে, কিন্তু তা সরাসরি ব্যবহার করতে দেয় না। এটি ব্যবহারের আগে টাইপ নিশ্চিত করা বাধ্যতামূলক, যা কোডকে নিরাপদ রাখে।


```

let value: unknown = "Hello";


// value.toUpperCase(); // এখানে TS এরর দিবে এবং আপনাকে আগে চেক করতে বলবে।```
