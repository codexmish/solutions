# TypeScript Generics

ডেভেলপার হিসেবে আমরা সবসময় এমন কোড লিখতে চাই যা বারবার ব্যবহার করা যায়। কিন্তু সমস্যা হয় যখন আমরা আলাদা আলাদা ডেটা স্ট্রাকচার (যেমন: `string`, `number`, বা `object`) নিয়ে কাজ করি। জেনারিক্স আমাদের এমন এক উপায় দেয় যাতে আমরা ফাংশন বা কম্পোনেন্ট বানানোর সময় নির্দিষ্ট টাইপ না বলে একটি "টাইপ ভেরিয়েবল" ব্যবহার করতে পারি। এতে কোড রিইউজেবল হয় এবং টাইপ সেফটিও বজায় থাকে।


## Generics কেন দরকার?

ধরুন আপনার এমন একটি ফাংশন দরকার যা ইনপুট হিসেবে যা পাবে, আউটপুট হিসেবেও তাই দিবে। আপনি যদি এটি any দিয়ে লেখেন, তবে আপনি টাইপ সেফটি হারাবেন। আর যদি শুধু number দিয়ে লেখেন, তবে এটি string-এর জন্য কাজ করবে না।

```
function identity(value: any): any {
  return value;
}
// এখানে টাইপ সেফটি নেই, 'any' ব্যবহারের কারণে রেজাল্ট কী আসবে TS তা জানে না।

```


## জেনারিক্স কিভাবে কাজ করে?

জেনারিক্স-এ আমরা এঙ্গেল ব্র্যাকেট <T> ব্যবহার করি। এখানে T হলো একটি প্লেসহোল্ডার বা টাইপ ভেরিয়েবল। যখন আমরা ফাংশনটি কল করি, তখন T সেই নির্দিষ্ট টাইপে রূপান্তরিত হয়।

```
function identity<T>(value: T): T {
  return value;
}

const num = identity<number>(100); // T এখানে number
const str = identity<string>("Hello"); // T এখানে string

```


## রিইউজেবল কম্পোনেন্ট এবং ইন্টারফেস

জেনারিক্স শুধু ফাংশনে নয়, ইন্টারফেস বা ক্লাসেও ব্যবহার করা যায়। এটি বড় প্রজেক্টে ডেটা স্ট্রাকচার হ্যান্ডেল করা অনেক সহজ করে দেয়।


```
interface Box<T> {
  content: T;
}

const stringBox: Box<string> = { content: "Next.js" };
const numberBox: Box<number> = { content: 2026 };

```

## উপসংহার
সহজ কথায়, জেনারিক্স আমাদের বারবার একই লজিক আলাদা আলাদা টাইপের জন্য লিখতে বাধা দেয়। এটি কোডকে ক্লিন রাখে এবং বড় স্কেলে যখন আমরা  ফ্রন্টএন্ড বা ব্যাকএন্ডে রিইউজেবল ফাংশন বানাবো, তখন এটি সবচেয়ে বড় হাতিয়ার হবে।
